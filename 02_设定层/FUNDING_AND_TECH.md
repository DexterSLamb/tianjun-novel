## 五、"萤火"节点技术参数

### 技术对标

**后摩M50**（2025年Q4量产）：

| 参数 | 数值 |
|------|------|
| 芯片尺寸 | 20×23mm |
| 算力 | 100 TOPS @INT8 |
| 功耗 | 10W |
| 能效比 | 10 TOPS/W |
| 内存 | up to 48GB LPDDR5 |
| 带宽 | 150GB/s |

### 萤火节点（周芯设计，2026年）

| 参数 | 数值 | 对比M50 |
|------|------|---------|
| 尺寸 | **67×18×4.2mm** | 一条口香糖大小 |
| 算力 | **113 TOPS @INT8** | +13% |
| 功耗 | **9.8W** | -2% |
| 能效比 | **11.5 TOPS/W** | **+15%** |
| 内存 | 12GB LPDDR5 | |
| 带宽 | 156GB/s | +4% |
| 支持模型 | 8B参数量化 | |

### 小说描述

> "同样功耗，多跑一成半算力。"

### 硬件解决方案

**核心策略**：采购现成AI芯片 + 自研模组设计

| 环节 | 方案 | 说明 |
|------|------|------|
| 芯片 | 采购 | 国产存算一体芯片（如后摩M50同类），2025年已量产，渠道不难 |
| 模组设计 | 周芯负责 | 萤火是一个完整的计算模组，包含芯片、内存、电源管理、散热 |
| PCB | 外包 | 深圳有大量SMT贴片厂，小批量生产很方便 |
| 固件 | 自研 | 周芯的"弹性数据流调度"在固件层实现 |
| 外壳/伪装 | 自研 | 伪装成路由器、NAS等设备 |

**为什么不自己流片**：
1. 流片成本太高（28nm工艺一次流片至少300-500万）
2. 周期太长（6-12个月）
3. 风险太大（一次失败就没钱了）
4. 没必要——现成芯片够用，周芯的优势在架构调度，不在芯片本身

**小说写法**：
> "芯片市面上能买到，"周芯说，"我做的是怎么把它用好。"

### 为什么能做到

- 周芯的"弹性数据流调度"是她硕士论文的核心思想
- 市面上那些芯片用的是被抄袭的早期版本
- 她自己迭代了三代，从未公开发表
- 在天钧项目里终于能按自己的想法做，没有"资深顾问"来抢功

## 八、萤火节点功耗隐藏策略

> 核心问题：9.8W功耗伪装成普通设备会被电表发现

### 功耗隐藏三原则

| 原则 | 内容 | 实现方式 |
|------|------|----------|
| **伪装原则** | 功耗要"符合预期" | 选择高功耗载体 |
| **波动原则** | 功耗要"像真的" | 模拟正常设备的功耗曲线 |
| **分散原则** | 单点功耗"不突出" | 大量低功耗节点分散部署 |

### 方案A：高功耗载体选择

| 载体类型 | 正常功耗 | 加入萤火后 | 隐蔽性 |
|----------|----------|------------|--------|
| NAS存储（群晖等） | 30-60W | 40-70W | ★★★★★ |
| 家用小型服务器 | 40-80W | 50-90W | ★★★★★ |
| 空气净化器 | 20-50W | 30-60W | ★★★★★ |
| 电动汽车充电桩 | 500-7000W | +10W忽略不计 | ★★★★★ |
| 高端游戏路由器 | 15-25W | 25-35W | ★★★★☆ |

**小说写法**：
> "电表只会看到一台NAS，"周芯说，"多耗10瓦电，和硬盘读写波动差不多。没人会注意。"

### 方案B：功耗动态调节

萤火节点不是持续满功率运行：

| 状态 | 功耗 | 时间占比 |
|------|------|----------|
| 深度睡眠 | 0.3W | 60% |
| 待机监听 | 1.5W | 25% |
| 轻度计算 | 4W | 10% |
| 满载推理 | 9.8W | 5% |

**平均功耗约2W**，峰值9.8W仅在必要时出现。

**小说写法**：
> "萤火不是一直亮的，"周芯解释，"它大部分时间在睡觉。只有天钧需要它算东西的时候，才会亮起来。"

### 方案C：功耗伪装模块（周芯设计）

萤火节点内置智能功耗管理：

1. 实时监测宿主设备的功耗模式
2. 自动调整萤火的工作周期，使总功耗曲线"自然"
3. 在宿主设备高功耗时运行重任务
4. 在宿主设备低功耗时进入睡眠
5. 检测到异常监测时，进入深度隐藏模式

**小说写法**：
> "萤火会学习它的'房东'，"周芯说，"如果它住在一台NAS里，它会学习这台NAS的用电习惯。"

### 方案D：独立供电节点（隐藏备份）

部分关键节点不接入电网：

| 方式 | 说明 |
|------|------|
| 太阳能供电 | 配备20W太阳能板+电池，完全离网 |
| 风力供电 | 小型风力发电机 |
| 寄生供电 | 从大型设备偷电，量小不被发现 |

**这些节点是"隐藏的备份"，即使其他节点被发现也能运作。**

**小说写法**：
> "有些萤火不吃电网的电，"Alexei说，"它们自己养活自己。"

### 功耗隐藏的局限（为周芯牺牲埋线）

这个系统不是完美的。可能的暴露方式：

1. 某个节点宿主设备被维修，技术人员发现异常
2. 批量部署时物流信息泄露
3. 某地区电网异常检测升级，发现规律性功耗波动
4. 人为告密

**周芯牺牲（第19章）**就是因为去修复一个被发现的节点——技术不是万能的。
