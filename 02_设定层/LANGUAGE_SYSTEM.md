## 六、禁止使用的表达

| 不要用 | 用这个代替 | 理由 |
|--------|------------|------|
| "觉醒" | **穿透** | "觉醒"太常见 |
| "真相" | **底码** | 更有技术感 |
| "命运" | **命轨** | 强调"被设计" |
| "努力" | **转轮** | 强调"无效" |
| "创造者/上帝" | **架构师** | 程序员视角 |
| "本能" | **原生** | 强调"预装" |

## 七、语言体系的设计逻辑

### 整体风格

- **简洁**：两个字为主
- **技术感**：程序员/工程师视角
- **冷峻**：不带感情色彩
- **画面感**：一听就有画面

### 递进关系

```
临界墙（你被挡住了）
    ↓
命轨（你的路是画好的）
    ↓
转轮（你的追求是空转）
    ↓
底码（你本身就是代码）
```

### 与《三体》的差异

| 《三体》 | 《天钧》 |
|----------|----------|
| 黑暗森林（宇宙法则） | 底码（人类本质） |
| 面壁者（对抗策略） | 穿透（认知状态） |
| 降维打击（外部攻击） | 转轮（内部困境） |

**核心差异**：《三体》的威胁来自外部，《天钧》的恐怖来自内部——你自己就是问题。

## 八、天钧输出格式规范

### 8.1 天钧的文字输出

使用 `>` markdown引用格式，每段之间用 `> ` 空行分隔：

```markdown
> 62.7%的人会在距离目标一步之遥时失败。
> 
> 这个比例在不同文化、不同时代、不同制度下几乎一致。
```

### 8.2 人类输入天钧的文字

同样使用 `>` 引用格式：

```markdown
他输入：

> 报告中的"设计"是什么含义？

回复在三秒后出现：

> 字面含义：某些人类特征的数学规律性超出了自然进化所能解释的范围...
```

### 8.3 禁止的格式

| 禁止 | 理由 |
|------|------|
| 「」直角引号 | 日式风格，与本书不符 |
| ""普通引号包裹长段 | 不够清晰，易与人物对话混淆 |

### 8.4 格式选择理由

1. **视觉区分**：`>` 引用格式在渲染后有明显缩进和左边框，一眼可识别为"机器输出"
2. **与人物对话区分**：人物对话用""，天钧输出用 `>`
3. **全书统一**：CH05、CH07已使用此格式，CH12及后续章节保持一致
